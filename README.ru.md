# docs-toolkit

Набор инструментов для документации: линтер/автофиксер Markdown, pre-commit хук, сниппеты VS Code, переименование скриншотов и автоматизация скриншотов через Selenium.

## Что внутри

| Инструмент | Что делает |
| --- | --- |
| [doccheck](#doccheck) | Линтер и автофиксер Markdown — проверка стиля, битых ссылок, правила из конфига |
| [Makefile](#makefile--быстрые-команды) | Короткие команды (`make check`, `make fix`) поверх doccheck |
| [pre-commit хук](#pre-commit-хук) | Запускает doccheck и сжимает PNG автоматически перед каждым коммитом |
| [VS Code сниппеты](#vs-code-сниппеты) | Блоки быстрого набора для примечаний, предупреждений, путей навигации, полей формы |
| [rename_images](#rename_images--переименование-скриншотов) | Переименовывает `imageN.png` после конвертации Word/Pandoc по подписям |
| [screenshots](#screenshots--автоматические-скриншоты) | Автоматические скриншоты через Selenium, с рабочим демо без реального приложения |

Подробное описание каждого инструмента — ниже.

## doccheck

Линтер и автофиксер Markdown-документации. Проверяет стиль, исправляет типовые ошибки и находит битые ссылки. Правила задаются в конфиг-файле — инструмент работает с любым проектом.

### Что умеет

- **Автоисправление** — заменяет буквы, знаки препинания, паттерны по правилам из конфига
- **Проверка стиля** — предупреждает о нарушениях, которые нельзя исправить автоматически
- **Проверка ссылок** — находит внутренние ссылки на несуществующие файлы
- **Цветной вывод** — ошибки красным, предупреждения жёлтым, исправления зелёным
- **CI-совместимость** — выход с кодом `1` при наличии ошибок

```text
Мой проект — проверено: 44 файлов

✗ docs/user-guide/payments/payment-order.md
  ! строка 12: Изображение без alt-текста
  ! строка 34: Тире после термина — заменить на двоеточие

✓ docs/user-guide/general-principles/lifecycle.md
  исправлено: ё → е (3×)
  исправлено: точка в заголовке примечания (1×)
```

**Что означают символы:**

| Символ | Значение |
| --- | --- |
| `✓` | Файл без проблем или уже исправлен |
| `✗` | В файле есть проблемы |
| `!` | Предупреждение — нужно посмотреть самому |
| `×` | Ошибка — нужно исправить вручную, блокирует CI |

**Пример на реальном проекте:** проверка двуязычной (RU/EN) технической документации — 38 Markdown-файлов, ~7 200 слов — полная проверка занимает меньше секунды, 0 битых внутренних ссылок, 0 изображений без alt-текста. (В этом проекте ещё не было скриншотов, поэтому `rename_images`/`screenshots` там не применялись.)

### Требования

- Python 3.11+ (зависимостей нет)
- Python 3.10: `pip install tomli`

### Быстрый старт

**1. Скопировать скрипт в проект:**

```bash
curl -O https://raw.githubusercontent.com/yourname/doc-tools/main/doccheck.py
```

**2. Создать конфиг:**

```bash
python3 doccheck.py --init
```

Откроется `doccheck.toml` — отредактируй правила под свой проект.

**3. Запустить проверку:**

```bash
python3 doccheck.py
```

### Команды

```bash
# Проверить всю документацию
python3 doccheck.py

# Автоматически исправить ошибки из раздела [[fix]]
python3 doccheck.py --fix

# Проверить конкретный файл или папку
python3 doccheck.py docs/user-guide/payments/

# Показывать только файлы с проблемами
python3 doccheck.py --quiet

# Указать другой конфиг
python3 doccheck.py --config path/to/custom.toml

# Создать пример конфига
python3 doccheck.py --init
```

### Конфигурация

Все правила хранятся в `doccheck.toml` рядом с проектом документации.

```toml
[project]
name   = "Название проекта"
docs   = "docs/"              # папка с Markdown-файлами
ignore = ["docs/archive/"]   # папки, которые пропускать

# Автоисправление: применяется при --fix
[[fix]]
pattern     = "ё"
replacement = "е"
description = "ё → е"

[[fix]]
pattern     = '!!! note "Примечание\."'
replacement = '!!! note "Примечание"'
description = "точка в заголовке примечания"

# Проверки: выводятся как предупреждения или ошибки
[[check]]
pattern     = '!\[\]\('
description = "Изображение без alt-текста"
severity    = "warn"   # или "error" — блокирует CI

[[check]]
pattern     = 'Перейдите в'
description = "Использовать «Откройте» вместо «Перейдите в»"
severity    = "warn"
```

#### Уровни severity

| Значение | Поведение |
| --- | --- |
| `warn` | Выводит предупреждение, не блокирует |
| `error` | Выводит ошибку, завершает с кодом 1 |

Правила из `[[fix]]` при проверке без `--fix` выводятся как `error`.

## Makefile — быстрые команды

Файл `Makefile` лежит в корне документационного проекта (не в tools/).

```bash
make install   # установить pre-commit хук (один раз)
make check     # проверить документацию
make fix       # исправить автоматически
```

На Mac `make` уже есть. На Windows нужен Git Bash.

## Pre-commit хук

Автоматически запускает проверку и сжимает PNG перед каждым коммитом.

```bash
# Установить (из корня проекта)
cp pre-commit .git/hooks/pre-commit
chmod +x .git/hooks/pre-commit
```

Требует установленного [oxipng](https://github.com/shssoichiro/oxipng) для сжатия PNG:

```bash
brew install oxipng   # macOS
```

## VS Code сниппеты

Файл `snippets.json` содержит готовые блоки для быстрого набора:

| Триггер | Что вставляет |
| --- | --- |
| `note` + Tab | Блок примечания `!!! note` |
| `warn` + Tab | Блок предупреждения `!!! warning` |
| `nav` + Tab | Путь навигации со стрелкой |
| `field` + Tab | Описание поля формы |
| `form` + Tab | Блок скриншот + поля формы |
| `anchor` + Tab | Заголовок с якорем `{ #id }` |
| `alink` + Tab | Ссылка на раздел другой страницы |
| `save` + Tab | Фраза подтверждения сохранения |

**Подключение в VS Code:**  
`Cmd+Shift+P` → Configure User Snippets → markdown → вставить содержимое `snippets.json`.

## rename_images — переименование скриншотов

После конвертации Word → Markdown через Pandoc все изображения называются `image1.png`, `image2.png` и т.д. Скрипт находит ближайшую подпись (или строку `Рис. N — ...`), предлагает читаемое имя на латинице, переименовывает файл и обновляет ссылки в .md.

```bash
# Посмотреть предложения без изменений
python3 rename_images.py --dry-run docs/

# Переименовать с подтверждением каждого имени
python3 rename_images.py docs/

# Принять все предложения автоматически
python3 rename_images.py --auto docs/

# Автоматически + удалить строки «Рис. N — ...»
python3 rename_images.py --auto --strip-captions docs/
```

В диалоге: **Enter** — принять, **своё имя** — ввести, **s** — пропустить, **Ctrl+C** — выйти.

Зависимостей нет — транслитерация встроена.

## screenshots — автоматические скриншоты

Автоматизация скриншотов через Selenium: открывает страницы приложения в браузере, снимает весь экран или один элемент, при необходимости сначала кликает до модального окна, сжимает PNG.

```bash
python3 screenshots.py --list           # список экранов из конфига
python3 screenshots.py --page payment   # снять один экран
python3 screenshots.py                  # снять все
```

Настройки страниц — в `screenshots.toml`:

```toml
[[pages]]
id          = "payment_order"
url         = "/payments/payment-order"
file        = "payment_order_form.png"
element     = ".form-container"   # снять только этот элемент, не весь экран
wait_for    = ".form-container"

[[pages]]
id          = "sign_modal"
url         = "/payments/payment-order/1"
file        = "sign_modal.png"
element     = ".modal-dialog"
actions     = [
  {click = "button.sign-btn"},    # кликнуть до модального окна перед снимком
  {wait  = ".modal-dialog"},
]
```

Требует Selenium и настоящих CSS-селекторов из вашего приложения:

```bash
pip install selenium
```

### Попробовать без реального приложения

В `tools/demo/` лежит самодостаточная страница логина + формы и конфиг под неё — можно увидеть, как инструмент реально работает, без какого-либо настоящего целевого приложения:

```bash
python3 -m http.server 8010 --directory tools/demo
python3 tools/screenshots.py --config tools/demo/screenshots.demo.toml
```

Скрипт залогинится, откроет демо-форму и сохранит скриншот только этого элемента в `tools/demo/output/`.

## Структура репозитория

```text
doccheck.py        — линтер и автофиксер стиля
rename_images.py   — переименование скриншотов после Word
screenshots.py     — автоматические скриншоты через Selenium
pre-commit         — git hook: doccheck + сжатие PNG
snippets.json      — сниппеты для VS Code
screenshots.toml   — конфиг экранов для screenshots.py
demo/              — самодостаточный пример для screenshots.py, без реального приложения
README.ru.md       — этот файл
README.md          — английская версия
```

`doccheck.toml` и `screenshots.toml` хранятся в каждом проекте отдельно.
